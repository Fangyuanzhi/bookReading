# 陪读 · 产品需求文档（PRD）

> 氛围陪伴式社交共读社区 · v0.1（MVP）
> 配套文档：技术规约 `TECH-SPEC.md`、数据库 `supabase-schema.sql`

---

## 1. 产品概述

一句话：一个让你**感到「不是一个人在读」**的小说阅读社区。

定位差异：

- 番茄 / 起点 = 孤独 + 功利的「刷文」。
- 微信读书 = 有社交，但偏工具。
- **陪读 = 把「氛围」与「陪伴」做成可感知的核心体验。**

核心命题：读书本不是一个人的事——用在场、段评、共读，把孤独的阅读变成「有人同行」。

## 2. 目标用户与场景

- **用户**：偏爱沉浸、安静、有共鸣感阅读的读者（非「爽文快消」用户）。
- **场景**：深夜独自读经典/原创，想知道「还有谁也读到这里、在这句话上想到了什么」；想读完能与人交流，而不是被算法推着刷。

## 3. 核心价值 / 差异化（护城河）

- **陪伴可开关**：结伴 / 独自一键切换——产品的灵魂控件。
- **在场感**：每章「此刻 N 人在读」、读者微光。
- **段评 / 划线想法**：他人的想法浮现在**原句旁**（锚定到具体句子，非整书评论区）。
- **氛围层**：暮 / 纸 / 夜主题、刻意的慢节奏、（后续）环境音。

> 差异化不在「能读 EPUB」，而在这套陪伴体验 + 社区沉淀。生成/渲染是入场券，陪伴才是壁垒。

## 4. 功能需求（按优先级）

### P0 · MVP 必做
- **账号**：注册 / 登录、资料（昵称、头像）。
- **书库**：公版书库浏览、搜索、书籍详情、继续阅读。
- **阅读器**：EPUB 渲染、翻页 / 滚动、字号、暮/纸/夜主题、进度记忆。
- **结伴 / 独自开关**。
- **段评 / 划线想法**：选中文字 → 写想法；他人公开想法锚定显示在原句旁；查看某段全部想法；点亮；私人笔记。
- **实时在场**：每章「N 人在读」。
- **实时段评**：新想法实时出现在对应段落。
- **章末书评**：发布、浏览、点亮。
- **内容合规**：避风港「通知—删除」入口、举报。

### P1
- **原创 UGC**：作者写作 / 章节管理 / 发布；原创书库。
- **个人主页**：我的书架、我的想法、被点亮。
- **轻社交**：关注 / 动态。
- **环境音**：雨声 / 壁炉 / 咖啡馆。

### P2
- **共读小组**：同书按进度共读、小组讨论、共读日历。
- **发现 / 推荐**。
- **原生 App**（P0 先用响应式 Web 覆盖全端）。

## 5. 非功能需求

- **全端响应式 Web**：一套 UI 适配手机 / 平板 / PC（P0 即满足「全端」诉求）。
- **性能**：章节首屏 < 2s；段评 / 在场实时延迟 < 1s。
- **内容合规**：版权红线（见技术规约 §7）；避风港流程。
- **隐私**：私人笔记仅本人可见；最小化数据采集。
- **可访问性**：键盘可达、足够对比度、可调字号、尊重 `prefers-reduced-motion`。

## 6. 范围与里程碑

- **MVP 明确不做**：原生 App、UGC 编辑器、共读小组、推荐、付费。
- **M1（MVP）**：公版书 + 阅读器 + 段评 + 在场 + 书评 + 主题 + 结伴/独自。
- **M2**：UGC + 个人主页 + 环境音。
- **M3**：共读小组 + 发现 + 商业化。

## 7. 关键指标

- **北极星**：周「陪伴阅读时长」（结伴模式下的阅读时长）。
- **辅助**：次留 / 周留、段评渗透率（读者中留过想法的比例）、人均段评 / 书评、被点亮数、共读完成率。

## 8. 风险与依赖

| 风险 | 说明 | 应对 |
|---|---|---|
| 内容版权（最高） | 商用托管盗版会被关站/起诉 | 只用公版 + 原创起步，避风港兜底，绝不盗版（详见技术规约 §7） |
| 冷启动 | 先有读者还是先有内容 | MVP 用公版经典破冷启动 |
| 段评锚定 | EPUB 重排导致锚点漂移 | CFI + 原文兜底（详见技术规约 §5.1） |
| 平台调性 | 反成瘾慢节奏 vs 增长压力 | 在设计上克制，不做无限刷 |
